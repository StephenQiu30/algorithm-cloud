# 排序算法综合对比

当学习完多种排序算法后，真正重要的问题不再是“某个算法怎么写”，而是“它和别的算法相比有什么区别”。本章以综合对比为核心，帮助学习者从全局视角理解不同排序算法的性能、稳定性和适用场景。

#### 八大排序算法的时间复杂度和空间复杂度如何对比？

八大排序算法的复杂度对比如下：

| 排序算法 | 最好时间 | 平均时间 | 最坏时间 | 空间复杂度 | 稳定性 |
|---------|---------|---------|---------|-----------|--------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 希尔排序 | O(n log n) | O(n^1.25) | O(n²) | O(1) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | 不稳定 |
| 基数排序 | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | 稳定 |

其中归并排序和堆排序的时间复杂度在三种情况下始终为 O(n log n)，是最稳定的；快速排序平均最快但最坏会退化；插入排序和冒泡排序在近乎有序时表现最好；基数排序在特定条件下可以突破 O(n log n) 的比较排序下界。

#### 如何根据数据特征选择合适的排序算法？

选择排序算法时应根据以下数据特征做判断：（1）**数据规模小（n < 50）**：使用插入排序，其常数因子小、实现简单，在小规模下优于递归算法。（2）**数据近乎有序**：使用插入排序或冒泡排序（加提前终止），它们在近乎有序数据上接近 O(n)。（3）**大规模随机数据**：使用快速排序，平均性能最优且缓存友好。（4）**需要最坏情况保证**：使用归并排序或堆排序，它们的最坏情况仍为 O(n log n)。（5）**需要稳定排序**：使用归并排序或插入排序；快速排序、堆排序和选择排序都不稳定。（6）**内存受限**：使用堆排序（O(1) 空间）或快速排序（O(log n) 栈空间），避免归并排序的 O(n) 额外空间。（7）**整数或定长字符串排序**：当数据范围可控时，基数排序可以达到 O(n) 线性时间。（8）**外部排序（数据大于内存）**：使用归并排序的多路归并变体。

#### 稳定排序和不稳定排序有什么区别？在什么场景下需要关注稳定性？

排序的稳定性是指：对于键值相等的元素，排序后它们的相对顺序是否与排序前一致。稳定的排序算法包括冒泡排序、插入排序、归并排序和基数排序；不稳定的包括选择排序、希尔排序、快速排序和堆排序。需要关注稳定性的典型场景：（1）**多字段排序**：先按价格排序再按销量排序，如果第二次排序不稳定，价格相同的商品在按销量排序后价格顺序会被打乱。稳定排序可以保证"销量相同时价格仍有序"。（2）**数据库 ORDER BY 多个字段**：SQL 引擎通常需要稳定排序以满足多字段排序语义。（3）**动画或 UI 渲染**：列表排序后保持相等元素不跳动，提供更好的用户体验。（4）**已有部分排序信息时**：利用前一次排序的结果做增量排序。

#### 比较排序和非比较排序的本质区别是什么？

比较排序（如冒泡、选择、插入、希尔、归并、快速、堆排序）通过元素之间的两两比较来确定相对大小关系。信息论证明，基于比较的排序算法的时间复杂度下界为 O(n log n)——因为 n 个元素有 n! 种排列，至少需要 log₂(n!) ≈ n log n 次比较才能区分所有排列。非比较排序（如计数排序、基数排序、桶排序）不依赖元素间的比较，而是利用元素的值域信息（如数值的位、范围）直接确定位置，因此可以突破 O(n log n) 的下界达到 O(n) 线性时间。但非比较排序有适用条件限制：需要知道数据的值域范围，且仅适用于特定数据类型（整数、定长字符串等），通用性不如比较排序。

#### 内部排序和外部排序有什么区别？各适合使用什么排序算法？

内部排序是指所有数据都能全部加载到内存中进行排序，八大排序算法都属于内部排序。外部排序是指数据量太大无法一次性全部放入内存，需要借助磁盘等外部存储进行排序，典型代表是多路归并排序。外部排序的基本流程：（1）将大文件分割为多个能放入内存的小块（Run）。（2）对每个小块在内存中使用内部排序算法（通常用快速排序或堆排序）排序。（3）将排序好的多个小块通过 K 路归并合并为最终结果，归并过程中使用最小堆高效选取 K 个块中的最小元素。优化手段包括：置换选择排序生成更长的初始 Run、败者树加速 K 路归并、双缓冲 I/O 减少磁盘等待。外部排序广泛应用于数据库索引构建、大数据离线处理中的 MapReduce Shuffle 阶段。

#### 在实际编程语言和框架中，标准库使用了什么排序算法？

主流编程语言的标准库排序实现如下：（1）**Java**：`Arrays.sort()` 对基本类型使用双轴快速排序（Dual-Pivot Quicksort），对对象类型使用 TimSort（基于归并排序的优化版，保证稳定性）。（2）**Python**：内置 `sort()` 和 `sorted()` 使用 TimSort。（3）**C++**：`std::sort()` 使用 IntroSort（快速排序 + 堆排序 + 插入排序的混合体），`std::stable_sort()` 使用归并排序。（4）**Go**：`sort.Sort()` 使用 Pattern-Defeating Quicksort（pdqsort），融合了快速排序和堆排序。（5）**Rust**：`sort()` 稳定排序使用 TimSort 变体，`sort_unstable()` 使用 pdqsort。这些实现都不是单一算法，而是多种排序算法的组合，根据数据规模和特征动态选择最优策略。

#### 本章小结

综合比较的价值在于帮助学习者建立“根据约束选算法”的思维方式，而不是机械记忆结论。学习本章后，应能根据数据规模、稳定性要求、内存限制和数据类型快速判断更合适的排序方案。
