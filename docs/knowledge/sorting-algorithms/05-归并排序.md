# 归并排序（Merge Sort）

#### 归并排序的基本原理是什么？

归并排序采用分治（Divide and Conquer）策略。它将序列递归地拆分为两个子序列，分别排序后再合并（Merge）。具体步骤为：（1）**分解**：将长度为 n 的序列从中间拆分为两个长度约为 n/2 的子序列。（2）**递归排序**：对两个子序列分别递归地应用归并排序。（3）**合并**：将两个已排序的子序列合并为一个有序序列。合并过程中，使用两个指针分别指向两个子序列的头部，每次选取较小的元素放入结果序列，直到所有元素合并完毕。递归的终止条件是子序列长度小于等于 1。

#### 归并排序是稳定排序吗？为什么？

归并排序是稳定的排序算法。稳定性的关键在于合并（Merge）步骤的实现：当两个子序列中出现值相等的元素时，始终优先选取左侧子序列的元素放入结果序列（即使用 `<=` 而非 `<` 作为比较条件）。由于左侧子序列中的元素在原始序列中位于右侧子序列之前，这种优先策略保证了相等元素的原始相对顺序被保留。这是归并排序相对于快速排序的一个重要优势——快速排序无法保证稳定性。

#### 归并排序的时间复杂度和空间复杂度分别是多少？

归并排序的时间复杂度在最好、最坏和平均情况下均为 O(n log n)。这是因为无论输入数据如何分布，归并排序都会将序列均匀分为两半（log n 层递归），每一层的合并操作总计需要 O(n) 时间。这种稳定的 O(n log n) 性能是归并排序相比快速排序的一大优势——快速排序在最坏情况下会退化为 O(n²)。空间复杂度为 O(n)，因为合并过程需要一个与原序列等大的辅助数组来暂存合并结果。递归调用栈的深度为 O(log n)，但辅助数组的 O(n) 是主导因素。

#### 归并排序有哪些优点和缺点？

**优点：** （1）时间复杂度始终为 O(n log n)，性能稳定可预测，不会因特定输入退化。（2）稳定的排序算法，适用于需要保持等值元素相对顺序的场景。（3）非常适合外部排序（External Sort），当数据量超过内存容量时，可以分块读入、排序、再多路归并。（4）天然适合并行化——两个子序列的排序可以并行执行。（5）适合链表排序，在链表上可以实现 O(1) 空间的归并排序。

**缺点：** （1）空间复杂度 O(n)，需要额外的辅助数组，不是原地排序。（2）在数组上实际运行速度通常慢于经过优化的快速排序（常数因子更大）。（3）递归实现中函数调用开销较大。（4）对于小规模数据，开销大于简单的 O(n²) 排序算法。

#### 归并排序有哪些常见的优化方法？

归并排序常见的优化方法包括：（1）**小规模子数组切换插入排序**：当子数组长度小于阈值（通常 16~64）时，使用插入排序替代递归归并排序，减少递归开销。这是 TimSort 采用的关键优化。（2）**避免不必要的合并**：如果左子数组的最大值已经小于等于右子数组的最小值，则两个子数组已经有序，无需合并操作。（3）**自底向上的迭代实现**：用循环代替递归，消除函数调用栈的开销，虽然代码稍复杂，但在某些场景下性能更好。（4）**原地归并排序**：通过旋转或块合并技术实现 O(1) 空间的归并排序，但实现复杂且常数因子大。（5）**多路归并**：在外部排序中使用 K 路归并减少 I/O 轮次。

#### 归并排序和快速排序应该如何选择？

归并排序和快速排序各有适用场景：（1）**稳定性要求**：需要稳定排序时选归并排序，快速排序不稳定。（2）**最坏情况性能**：要求最坏情况也是 O(n log n) 时选归并排序，快速排序最坏为 O(n²)。（3）**内存限制**：内存紧张时选快速排序（O(log n) 栈空间），归并排序需要 O(n) 额外空间。（4）**平均性能**：在数组上的平均性能，经过优化的快速排序通常优于归并排序（缓存友好性更好）。（5）**外部排序**：数据无法全部载入内存时选归并排序。（6）**链表排序**：链表上归并排序是首选，快速排序在链表上效率较差。Java 对对象数组使用 TimSort（基于归并排序），对基本类型数组使用双轴快速排序。
