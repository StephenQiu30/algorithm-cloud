# 堆排序（Heap Sort）

#### 堆排序的基本原理是什么？

堆排序利用"堆"这种数据结构进行排序。堆是一棵完全二叉树，分为最大堆（每个节点的值大于等于其子节点）和最小堆（每个节点的值小于等于其子节点）。堆排序的过程分两个阶段：（1）**建堆**：将无序数组原地构建为一个最大堆，从最后一个非叶子节点开始，逐个执行"下沉"（Sift Down）操作，时间复杂度为 O(n)。（2）**排序**：反复将堆顶（最大值）与堆的末尾元素交换，然后将堆大小减 1 并对新的堆顶执行下沉操作以恢复堆性质。每次操作确定一个元素的最终位置，经过 n-1 次交换后，数组变为升序。整个堆排序利用了数组下标即可表示父子关系的特性，不需要额外的指针。

#### 堆排序是稳定排序吗？为什么？

堆排序是不稳定的排序算法。不稳定性的原因有两个：（1）在建堆过程中，下沉操作会改变相等元素的相对顺序。（2）在排序阶段，堆顶元素与堆末尾元素交换时，可能将一个值相等的元素从前面交换到后面。例如数组 `[3a, 3b, 2]` 建最大堆后可能变为 `[3a, 3b, 2]` 或 `[3b, 3a, 2]`，排序过程中 3a 和 3b 的相对位置可能被打破。堆的树结构本身就不适合保持相等元素的线性相对顺序。

#### 堆排序的时间复杂度和空间复杂度分别是多少？

堆排序在最好、最坏和平均情况下的时间复杂度均为 O(n log n)。建堆阶段的时间复杂度为 O(n)（通过数学证明，自底向上建堆的总操作次数不超过 2n）。排序阶段需要 n-1 次交换和下沉操作，每次下沉的最大深度为 O(log n)，因此排序阶段为 O(n log n)。总时间复杂度由排序阶段主导，为 O(n log n)。空间复杂度为 O(1)，堆排序是原地排序算法，仅需一个临时变量用于交换，不需要额外的辅助数组。

#### 堆排序有哪些优点和缺点？

**优点：** （1）最坏情况时间复杂度稳定在 O(n log n)，不会像快速排序那样退化到 O(n²)，具有确定性的性能保证。（2）空间复杂度 O(1)，是原地排序，无需额外数组。（3）是唯一同时满足"最坏 O(n log n)"和"空间 O(1)"两个条件的比较排序算法。（4）非常适合求 Top-K 问题——用大小为 K 的堆即可在 O(n log K) 时间内找到前 K 大/小的元素。

**缺点：** （1）不稳定排序。（2）缓存不友好——堆的父子节点在数组中的下标关系（`2i+1` 和 `2i+2`）导致跳跃式内存访问，CPU 缓存命中率低。（3）实际运行速度通常慢于快速排序（同为 O(n log n)，但常数因子更大）。（4）不适合近乎有序的数据——无法像插入排序那样利用已有序性。（5）建堆过程会打乱原有数据的大部分顺序。

#### 堆排序在实际工程中有哪些典型应用？

堆排序和堆数据结构在工程中有广泛应用：（1）**Top-K 问题**：在海量数据中找到最大/最小的 K 个元素，维护一个大小为 K 的堆，时间复杂度 O(n log K)，空间仅需 O(K)。（2）**优先队列**：Java 的 `PriorityQueue`、C++ 的 `priority_queue` 底层均基于堆实现，用于任务调度、Dijkstra 最短路径算法、Huffman 编码等。（3）**IntroSort 的兜底**：C++ STL 的 `std::sort` 在快速排序递归深度过大时切换为堆排序，利用其 O(n log n) 最坏保证避免退化。（4）**定时器管理**：操作系统和网络框架使用最小堆管理定时任务，高效获取最近到期的任务。（5）**多路归并**：外部排序中使用堆进行 K 路归并，从 K 个有序块中每次取最小值。

#### 堆排序和快速排序应该如何选择？

堆排序和快速排序的选择取决于具体需求：（1）**性能追求**：追求平均最快速度选快速排序，其缓存友好性和较小的常数因子使其在大多数情况下更快。（2）**最坏保证**：要求确定性的 O(n log n) 保证选堆排序，特别是在安全敏感场景中防止恶意输入导致的排序退化攻击。（3）**空间限制**：两者都是原地排序，但堆排序的 O(1) 优于快速排序的 O(log n) 栈空间。（4）**数据特征**：重复元素多时选三路快排；需要 Top-K 时选堆排序。（5）**最佳实践**：现代标准库通常使用 IntroSort，结合两者优点——先快排，深度过大时切堆排，小规模切插入排序。
