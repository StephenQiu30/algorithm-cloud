# 插入排序（Insertion Sort）

#### 插入排序的基本原理是什么？

插入排序的核心思想类似于整理扑克牌：将序列分为已排序和未排序两部分，每次从未排序部分取出一个元素，在已排序部分中从后往前逐一比较，找到该元素应该插入的位置，将其插入到正确位置。具体地，初始时认为第一个元素已经有序，从第二个元素开始，将其与已排序部分的元素逐一比较，大于该元素的元素依次后移一位，直到找到不大于该元素的位置或到达序列头部，然后将元素插入到空出的位置。

#### 插入排序是稳定排序吗？为什么？

插入排序是稳定的排序算法。在比较过程中，当待插入元素与已排序部分的某个元素相等时，插入排序不会将待插入元素移动到该相等元素的前面——因为比较条件通常使用严格大于（`>`），相等时不再后移，待插入元素被放置在相等元素的后面。这就保证了值相同的元素在排序前后的相对位置不会改变，满足了排序稳定性的定义。

#### 插入排序的时间复杂度和空间复杂度分别是多少？

插入排序的最好时间复杂度为 O(n)，当输入序列已经有序时，每个元素只需与前一个元素比较一次即可确认位置，总共需要 n-1 次比较。最坏时间复杂度为 O(n²)，当输入序列完全逆序时，每个元素都需要与已排序部分的所有元素比较并移动。平均时间复杂度为 O(n²)。空间复杂度为 O(1)，插入排序是原地排序算法，只需一个临时变量来暂存当前待插入的元素。

#### 插入排序有哪些优点和缺点？

**优点：** （1）实现简单，代码易于编写和维护。（2）稳定排序，保持等值元素的相对顺序。（3）对于小规模数据或近乎有序的数据效率极高，最好情况 O(n)。（4）自适应性强——输入越接近有序，性能越好。（5）常被作为高级排序算法的子程序，如 TimSort 和 IntroSort 在小规模子数组上使用插入排序。（6）在线算法：可以在接收到数据的同时逐步排序，不需要预先知道全部数据。

**缺点：** （1）最坏和平均时间复杂度为 O(n²)，不适合大规模乱序数据。（2）数据移动次数较多，每次插入可能需要移动大量元素。（3）对逆序数据极不友好，性能会退化到最坏情况。

#### 插入排序与冒泡排序、选择排序相比有什么优势？

在三种 O(n²) 排序算法中，插入排序通常是实际性能最好的：（1）**自适应性**：插入排序能利用输入数据的已有序性，对于近乎有序的数据接近 O(n)，而选择排序无法利用这一特点。（2）**数据移动方式**：插入排序使用"移动"操作而非"交换"操作，每次移动只需一次赋值，而冒泡排序的交换需要三次赋值，因此常数因子更小。（3）**缓存友好性**：插入排序按顺序访问数据，对 CPU 缓存更友好。（4）**稳定性**：插入排序和冒泡排序都是稳定的，选择排序不稳定。综合来看，当必须使用简单排序算法时，插入排序通常是最优选择。

#### 插入排序在实际工程中有哪些应用？

插入排序在实际工程中有广泛应用：（1）**TimSort 的组成部分**：Python 和 Java 的默认排序算法 TimSort 使用插入排序对小于 64 的子数组进行排序。（2）**IntroSort 的兜底**：C++ STL 的 `std::sort` 使用 IntroSort，当递归深度过大或子数组足够小时切换为插入排序。（3）**小规模数据排序**：当数据量小于某个阈值（通常 16~64）时，插入排序的常数因子优势使其快于递归开销较大的快速排序。（4）**在线数据流排序**：需要对流式数据进行实时排序维护时，插入排序的在线特性非常适合。
