# 快速排序（Quick Sort）

#### 快速排序的基本原理是什么？

快速排序基于分治策略，其核心操作是"分区"（Partition）。具体步骤为：（1）**选取基准值（Pivot）**：从序列中选择一个元素作为基准值。（2）**分区操作**：将序列重新排列，使得所有小于基准值的元素放在基准值左边，所有大于基准值的元素放在基准值右边，基准值被放到最终正确的位置。（3）**递归排序**：对基准值左右两边的子序列递归地应用快速排序。与归并排序"先分后合"不同，快速排序是"先排后分"——在分区过程中就完成了元素的大致排序，不需要合并步骤。

#### 快速排序是稳定排序吗？为什么？

快速排序是不稳定的排序算法。不稳定性源于分区操作中的元素交换。在 Partition 过程中，当元素与基准值比较并交换位置时，相等的元素可能因为与不同位置的元素交换而改变相对顺序。例如 `[4a, 3, 4b, 2]`，以最后一个元素 2 为 Pivot 进行分区后，4a 和 4b 的相对位置可能被打破。虽然可以通过特殊的分区方案（如三向分区）或使用额外空间实现稳定版本，但标准的原地快速排序是不稳定的。

#### 快速排序的时间复杂度和空间复杂度分别是多少？

快速排序的平均时间复杂度为 O(n log n)，这是因为每次分区大致将序列均等分为两部分，递归深度为 O(log n)，每层处理 O(n) 个元素。最好情况也是 O(n log n)。最坏时间复杂度为 O(n²)，当每次选择的 Pivot 都是当前子序列的最大或最小元素时（如对已排序序列总选第一个元素做 Pivot），分区极度不均匀导致退化。空间复杂度为 O(log n)，即递归调用栈的平均深度；最坏情况下栈深度为 O(n)。快速排序本身是原地排序，不需要额外的数据数组。

#### 快速排序有哪些优点和缺点？

**优点：** （1）平均情况下是最快的通用排序算法之一，常数因子小于归并排序。（2）原地排序，空间效率高，仅需 O(log n) 栈空间。（3）缓存友好性好，局部性访问使得 CPU 缓存命中率高。（4）易于优化，有大量成熟的优化变体可选。（5）实际基准测试中几乎总是快于归并排序（在数组上）。

**缺点：** （1）最坏情况 O(n²)，虽然可以通过随机化 Pivot 降低概率，但无法完全避免。（2）不稳定排序。（3）递归实现中栈溢出风险，最坏情况递归深度为 O(n)。（4）对于有大量重复元素的数据，标准二路快排性能较差（需使用三路快排优化）。

#### 快速排序有哪些常见的优化方法？

快速排序有多种经典优化方案：（1）**随机化 Pivot**：随机选取基准值而非固定选第一个或最后一个元素，可以有效避免对特定数据分布的最坏情况，将最坏情况概率降到极低。（2）**三数取中法（Median-of-Three）**：取序列首、中、尾三个元素的中位数作为 Pivot，进一步平衡分区。（3）**三路快排（Three-Way Partition）**：将序列分为小于、等于、大于 Pivot 的三部分，对含大量重复元素的数据效果显著，荷兰国旗问题方案即为此。（4）**尾递归优化**：对较大的子序列使用迭代、较小的子序列使用递归，将栈空间降到 O(log n)。（5）**小规模切换插入排序**：当子数组小于阈值（通常 10~20）时，切换为插入排序。（6）**IntroSort**：结合快速排序、堆排序和插入排序，当递归深度超过 2log₂n 时切换为堆排序，兼顾平均性能和最坏保证。C++ STL 的 `std::sort` 即采用此方案。

#### 快速排序为什么在实际应用中比归并排序更受欢迎？

快速排序在实际工程中更受欢迎主要有以下原因：（1）**缓存友好性**：快速排序在连续内存上顺序访问数据，CPU 缓存命中率高；而归并排序需要在原数组和辅助数组之间来回复制，缓存利用率较低。（2）**空间效率**：快速排序是原地排序只需 O(log n) 栈空间，归并排序需要 O(n) 辅助空间。（3）**常数因子小**：快速排序的内循环（比较和交换）操作非常简单高效。（4）**优化成熟**：IntroSort 等变体已经消除了最坏情况的风险。然而在需要稳定性或外部排序的场景中，归并排序仍是更好的选择。
