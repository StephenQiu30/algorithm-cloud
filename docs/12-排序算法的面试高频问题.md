# 排序算法的面试高频问题

面试中的排序问题通常不会只考“写出某个排序代码”，而是更关注复杂度分析、场景选择、边界处理和工程意识。本章将常见高频问题集中整理，帮助学习者把排序知识转化为更具表达力的面试答案。

#### 如何在 O(n) 时间内对一个包含大量重复值的数组排序？

当数组中元素的值域范围有限（如年龄 0~150、成绩 0~100）时，可以使用计数排序在 O(n) 时间内完成排序。具体步骤：（1）统计每个值出现的次数，存入计数数组 `count[value]`。（2）对计数数组做前缀和，`count[i]` 表示值小于等于 i 的元素总数，即值 i 在排序结果中的最后位置。（3）从后向前遍历原数组，将每个元素放到计数数组指示的位置，同时将对应计数减 1。从后向前遍历保证了稳定性。时间复杂度 O(n + k)（k 为值域范围），空间复杂度 O(n + k)。如果值域很大（如 int 范围），计数排序不适用，此时应考虑三路快排或基数排序。

#### 如何找到无序数组中第 K 大的元素？

找第 K 大元素有以下方法：（1）**快速选择算法（Quickselect）**：基于快速排序的 Partition 思想，每次分区后只递归包含第 K 大元素的那一半，平均时间 O(n)、最坏 O(n²)、空间 O(1)。这是时间复杂度最优的方法。（2）**堆排序思想**：维护一个大小为 K 的最小堆，遍历数组，若元素大于堆顶则替换并调整堆。遍历完后堆顶即为第 K 大元素。时间 O(n log K)、空间 O(K)，适合数据流场景。（3）**BFPRT 算法（Median of Medians）**：Quickselect 的改进版，通过精心选择 Pivot 保证最坏也是 O(n)，但常数因子较大，实际中较少使用。Java 中可以使用 `PriorityQueue` 实现堆方案。这是 LeetCode 第 215 题。

#### 如何合并 K 个有序数组或链表？

合并 K 个有序序列是归并排序的扩展应用，有以下方法：（1）**最小堆（优先队列）K 路归并**：将 K 个序列的头元素放入最小堆中，每次取出堆顶（全局最小），将其所在序列的下一个元素入堆。时间 O(N log K)（N 为总元素数），空间 O(K)。这是最高效的方法。（2）**分治归并**：类似归并排序，将 K 个序列两两归并，递归合并直到只剩一个序列。时间同样 O(N log K)，但常数因子稍大。（3）**逐一归并**：将第 1 个和第 2 个合并，结果再与第 3 个合并，依次进行。时间 O(NK)，效率最低，不推荐。Java 的 `PriorityQueue` 配合自定义 Comparator 可以简洁实现堆方案。这是 LeetCode 第 23 题。

#### 如何判断一个排序算法实现是否正确？有哪些测试方法？

验证排序算法的正确性可以使用以下测试策略：（1）**边界用例**：空数组、只有一个元素、两个元素（有序/逆序/相等）。（2）**特殊分布**：已有序数组、完全逆序数组、所有元素相同、仅两种不同值。（3）**随机测试（对拍法）**：生成大量随机数组，用自己的排序结果与标准库的 `Arrays.sort()` 结果比较，不一致则发现 Bug。（4）**稳定性测试**：构造含重复值的数组，排序后检查等值元素的相对顺序是否保持不变。（5）**大规模压力测试**：对百万级数据排序，检验是否存在性能退化、栈溢出或超时。（6）**逆序对计数检验**：排序后的数组逆序对数应为 0，可以用归并排序计算逆序对来验证。（7）**有序性断言**：遍历排序结果，检查 `a[i] <= a[i+1]` 对所有 i 成立。

#### 在面试中手写快速排序需要注意哪些关键点？

面试中手写快速排序的关键注意点：（1）**Pivot 选取**：不要固定选第一个或最后一个元素，使用随机化或三数取中法，向面试官说明理由。（2）**分区方法选择**：推荐 Lomuto 分区（单指针，逻辑简单）或 Hoare 分区（双指针，交换次数少）。（3）**等于 Pivot 的元素处理**：明确说明是放左边还是右边，如果面试官追问重复元素场景，展示三路快排方案。（4）**递归终止条件**：`if (left >= right) return;` 不要遗漏。（5）**整数溢出**：`mid = left + (right - left) / 2` 而非 `(left + right) / 2`。（6）**优化加分点**：提到小规模切换插入排序、尾递归优化、IntroSort 等实际工程优化。（7）**复杂度分析**：能够清晰分析平均 O(n log n) 和最坏 O(n²) 的原因。正确处理这些细节体现了对算法的深入理解而非死记硬背。

#### 什么是排序的"原地"特性？为什么它在实际应用中很重要？

原地排序（In-Place Sort）是指排序过程中只使用 O(1) 或 O(log n) 的额外空间，不需要与输入规模成正比的辅助存储。原地排序算法包括冒泡排序、选择排序、插入排序、希尔排序、堆排序和快速排序。非原地排序包括归并排序（O(n) 辅助数组）和基数排序（O(n+k) 辅助空间）。原地特性在以下场景中尤为重要：（1）**嵌入式系统**：内存极度受限（如 KB 级），无法分配大块辅助空间。（2）**实时系统**：动态内存分配可能引入不可预测的延迟（GC 暂停），原地排序避免了这个问题。（3）**大数据场景**：当数据刚好能放入内存时，原地排序节省的空间可以用来处理更多数据。（4）**缓存效率**：原地排序在单一数组上操作，数据局部性好，缓存命中率高。堆排序是唯一同时满足"最坏 O(n log n)"和"O(1) 空间"的比较排序算法。

#### 本章小结

面试中的排序题考察的不只是算法记忆，更考察你是否理解“为什么这样选、这样写、这样分析”。学习本章后，建议继续训练把复杂度、稳定性、原地性和适用场景放到同一个回答框架里表达清楚。
