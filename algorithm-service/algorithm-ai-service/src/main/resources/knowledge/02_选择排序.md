# 选择排序 (Selection Sort) 深度解析

选择排序（Selection Sort）是一种简单直观的排序算法。它的工作原理是：首先在未排序序列中找到最小（大）元素，存放到排序序列的起始位置，然后，再从剩余未排序元素中继续寻找最小（大）元素，然后放到已排序序列的末尾。

## 算法原理

1. 初始状态：无序区为 `R[1..n]`，有序区为空。
2. 第 $i$ 趟排序开始时，当前有序区和无序区分别为 `R[1..i-1]` 和 `R[i..n]`。该趟排序从当前无序区中选出关键字最小的记录
   `R[k]`，将它与无序区的第 1 个记录 `R[i]` 交换。
3. $n-1$ 趟以后，所有记录均已排好序。

## 性能分析

- **时间复杂度 (平均)**: $O(n^2)$
- **时间复杂度 (最坏)**: $O(n^2)$
- **时间复杂度 (最好)**: $O(n^2)$ - 无论初始状态如何，比较次数都是固定的。
- **空间复杂度**: $O(1)$
- **稳定性**: **不稳定** - 例如 `[5, 8, 5, 2, 9]`，第一轮会将第一个 `5` 与 `2` 交换，导致两个 `5` 的相对位置改变。

## 优缺点

- **优点**: 移动数据的次数非常少（最多 $n-1$ 次），如果交换操作的开销远大于比较操作，选择排序性能可能优于某些 $O(n^2)$ 算法。
- **缺点**: 比较次数多，效率低。

## Java 实现

```java
public class SelectionSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            int minIndex = i;
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[minIndex]) {
                    minIndex = j;
                }
            }
            if (minIndex != i) {
                int temp = arr[i];
                arr[i] = arr[minIndex];
                arr[minIndex] = temp;
            }
        }
    }
}
```

## 适用场景

- 数据规模较小。
- 对稳定性没有要求。
- 内存资源受限，且希望最大程度减少数据移动（写入次数）的场景。
