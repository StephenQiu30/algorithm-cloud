# 堆排序 (Heap Sort) 深度解析

堆排序（Heap Sort）是指利用“堆”这种数据结构所设计的一种排序算法。堆是一个近似完全二叉树的结构，并同时满足堆积性质：父结点的键值总是大于或等于（在最大堆中）任何子结点的键值。

## 算法原理
1. **构造堆**: 将初始待排序关键字序列 `(R1, R2, ..., Rn)` 构建成大顶堆。
2. **交换与调整**: 将堆顶元素 `R[1]` 与最后一个元素 `R[n]` 交换，此时得到新的无序区 `(R1, R2, ..., Rn-1)` 和新的有序区 `(Rn)`。
3. **重新堆化**: 由于交换后新的堆顶 `R[1]` 可能违反堆性质，因此需要对当前无序区 `(R1, R2, ..., Rn-1)` 进行调整，使其重新变成大顶堆。
4. **循环**: 重复步骤 2 和 3，直到有序区的元素个数为 $n-1$。

## 性能分析
- **时间复杂度 (平均/最好/最坏)**: 均为 $O(n \log n)$。
- **空间复杂度**: $O(1)$ - 原地排序。
- **稳定性**: **不稳定** - 在调整堆的过程中，相同元素的相对位置可能会改变。

## 优缺点
- **优点**: 时间复杂度稳定在 $O(n \log n)$。空间占用极低（原地排序）。
- **缺点**: 局部性较差，对于已经有序的序列，由于需要重新建堆，效率并不见得比插入排序高。

## Java 实现
```java
public class HeapSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        int n = arr.length;

        // 1. 构建最大堆 (从最后一个非叶子节点开始向上调整)
        for (int i = n / 2 - 1; i >= 0; i--) {
            heapify(arr, n, i);
        }

        // 2. 交换堆顶元素与末尾元素，并重新调整堆
        for (int i = n - 1; i > 0; i--) {
            int temp = arr[0];
            arr[0] = arr[i];
            arr[i] = temp;
            heapify(arr, i, 0); // 调整剩余部分
        }
    }

    private static void heapify(int[] arr, int n, int i) {
        int largest = i; 
        int left = 2 * i + 1;
        int right = 2 * i + 2;

        if (left < n && arr[left] > arr[largest]) largest = left;
        if (right < n && arr[right] > arr[largest]) largest = right;

        if (largest != i) {
            int swap = arr[i];
            arr[i] = arr[largest];
            arr[largest] = swap;
            // 递归调整子树
            heapify(arr, n, largest);
        }
    }
}
```

## 适用场景
- 数据规模非常大且对内存有严格限制。
- 求解 Top-K 问题（例如在一亿个数字中找到最大的一百个）。
- 实时系统，因其最坏情况下的表现非常稳定。
