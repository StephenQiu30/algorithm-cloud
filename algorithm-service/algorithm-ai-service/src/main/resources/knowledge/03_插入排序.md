# 插入排序 (Insertion Sort) 深度解析

插入排序（Insertion Sort）的工作原理是通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。插入排序在实现上，通常采用 in-place 排序（即只需用到 $O(1)$ 的额外空间的排序）。

## 算法原理
1. 从第一个元素开始，该元素可以认为已经被排序。
2. 取出下一个元素，在已经排序的元素序列中从后向前扫描。
3. 如果该元素（已排序）大于新元素，将该元素移到下一位置。
4. 重复步骤 3，直到找到已排序的元素小于或者等于新元素的位置。
5. 将新元素插入到该位置后。
6. 重复步骤 2~5。

## 性能分析
- **时间复杂度 (平均)**: $O(n^2)$
- **时间复杂度 (最坏)**: $O(n^2)$ - 反序情况。
- **时间复杂度 (最好)**: $O(n)$ - 数组已经是有序的情况。
- **空间复杂度**: $O(1)$
- **稳定性**: **稳定** - 相同元素插入时会放在相等元素的后面。

## 优缺点
- **优点**: 对于近乎有序的数组效率极高。是稳定性排序。
- **缺点**: 大规模无序数据时性能较差。

## Java 实现
```java
public class InsertionSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        int n = arr.length;
        for (int i = 1; i < n; i++) {
            int target = arr[i];
            int j = i - 1;
            while (j >= 0 && arr[j] > target) {
                arr[j + 1] = arr[j];
                j--;
            }
            arr[j + 1] = target;
        }
    }
}
```

## 适用场景
- 数据量小。
- 数组基本有序（如流式数据处理）。
- 作为快速排序等复杂算法的底层优化细节（当子序列长度达到某个阈值时改用插入排序，例如 JDK 中的 `DualPivotQuicksort`）。
