# 希尔排序 (Shell Sort) 深度解析

希尔排序（Shell Sort）是插入排序的一种更高效的改进版本，也称为“缩小增量排序”。它是第一个突破 $O(n^2)$ 时间复杂度的排序算法。它通过比较相距一定间隔的元素来工作，各阶段使用的间隔随着算法的进行而减小，直到只比较相邻元素的最后一趟排序为止。

## 算法原理
1. 选择一个增量序列 $t_1, t_2, ..., t_k$，其中 $t_i > t_{j}$，$t_k = 1$。
2. 按增量序列个数 $k$，对序列进行 $k$ 趟排序。
3. 每趟排序，根据对应的增量 $t_i$，将待排序列分割成若干长度为 $m$ 的子序列，分别对各子表进行直接插入排序。
4. 当增量减小为 1 时，整个序列恰好被分成一个子序列，最后完成一次直接插入排序。

## 性能分析
- **时间复杂度 (平均)**: $O(n \log n)$ 到 $O(n^2)$ 之间（取决于增量序列的选择）。常用的 $gap/2$ 序列约为 $O(n^{1.3})$ 或 $O(n^{1.5})$。
- **时间复杂度 (最坏)**: $O(n^2)$ (对于某些增量序列)。
- **空间复杂度**: $O(1)$
- **稳定性**: **不稳定** - 在不同的增量分组中，相同元素的相对位置可能会改变。

## 优缺点
- **优点**: 比普通的插入排序快得多，且无需额外大量空间。
- **缺点**: 增量序列的选择会显著影响性能，且不稳定。

## Java 实现 (希尔增量序列)
```java
public class ShellSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        int n = arr.length;
        // gap 为增量，每次减半
        for (int gap = n / 2; gap > 0; gap /= 2) {
            // 对每个子序列进行插入排序
            for (int i = gap; i < n; i++) {
                int target = arr[i];
                int j = i - gap;
                while (j >= 0 && arr[j] > target) {
                    arr[j + gap] = arr[j];
                    j -= gap;
                }
                arr[j + gap] = target;
            }
        }
    }
}
```

## 适用场景
- 中等规模的数据排序。
- 虽然现在的快排、归并更常用，但在嵌入式等对内存极其敏感且数据量不是特别巨大的场景下仍有价值。
