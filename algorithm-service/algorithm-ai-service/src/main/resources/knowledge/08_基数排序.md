# 基数排序 (Radix Sort) 深度解析

基数排序（Radix Sort）属于“分配式排序”（Distribution Sort），是非比较型整数排序算法。它的原理是将整数按位数切割成不同的数字，然后按每个位数分别比较。由于整数也可以表达字符串（例如名字或日期）和特定格式的浮点数，所以基数排序也不是只能使用于整数。

## 算法原理
1. 取得数组中的最大数，并取得位数（根据最大值确定需要进行多少轮分配）。
2. 从最低位（个位）开始，根据该位上的数字将所有元素分配到 10 个“桶”中。
3. 按照桶的顺序以及桶内元素的放入顺序，把元素重新收集回原数组。
4. 对下一位（十位、百位...）重复步骤 2 和 3，直到达到最大位数。

## 性能分析
- **时间复杂度**: $O(d \cdot (n + r))$，其中 $d$ 是位数，$r$ 是基数（例如 10 进制中 $r=10$）。当 $d$ 较小时，性能优于 $O(n \log n)$。
- **空间复杂度**: $O(n + r)$ - 需要额外的桶存储。
- **稳定性**: **稳定** - 个位的相对顺序会在十位排序时被保留。

## 优缺点
- **优点**: 对特定范围的数据（如多位整数）速度极快。是稳定性排序。
- **缺点**: 受限于数据类型（必须能映射到离散的桶）。空间开销相比原地排序较大。

## Java 实现
```java
import java.util.*;

public class RadixSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        int max = Arrays.stream(arr).max().getAsInt();
        
        // 从个位开始，指数 exp 以 10 的倍数增长
        for (int exp = 1; max / exp > 0; exp *= 10) {
            countSort(arr, exp);
        }
    }

    private static void countSort(int[] arr, int exp) {
        int n = arr.length;
        int[] output = new int[n];
        int[] count = new int[10];

        // 统计出现次数
        for (int i = 0; i < n; i++) {
            count[(arr[i] / exp) % 10]++;
        }

        // 计算累加位置
        for (int i = 1; i < 10; i++) {
            count[i] += count[i - 1];
        }

        // 构建输出数组（从后往前扫描以保证稳定性）
        for (int i = n - 1; i >= 0; i--) {
            int digit = (arr[i] / exp) % 10;
            output[count[digit] - 1] = arr[i];
            count[digit]--;
        }

        // 拷贝回原数组
        System.arraycopy(output, 0, arr, 0, n);
    }
}
```

## 适用场景
- 数据长度相差不大且数值范围有限（如电话号码、身份证号）。
- 排序整数、字符串等可以拆分为“位”的数据。
- 需要极高性能且内存充裕的大数据处理流程。
